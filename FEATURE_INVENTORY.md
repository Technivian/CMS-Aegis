## Notes

- `Done` means the feature is present and usable in the codebase.
- `Partially done` means the feature exists, but live proof, UX depth, or edge-case handling is still missing.
- `Unknown` would be used where the codebase does not expose enough evidence to prove the feature.
- `Broken` is reserved for features that are present but currently fail; I did not find a broad set of hard-broken customer flows in the current checkout.
