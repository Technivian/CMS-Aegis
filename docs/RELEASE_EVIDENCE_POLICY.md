# Release Evidence Policy

Last updated: 2026-04-18

## Policy

No production release candidate is approved unless all required evidence artifacts are attached to the release workflow run.

## Required Evidence Bundle

Generated by GitHub Actions workflow: `.github/workflows/release-candidate-evidence.yml`

Artifact contents:

- `manage-check.txt`
- `cross-tenant-tests.txt`
- `performance-guardrails.txt`
- `alert-evaluation.json`
- `route-latency-core.json`
- `route-latency-auth.json`
- `load-test-2x.json`
- `query-plan.md`
- `metadata.txt`

Additional required operations evidence (Sprint 3 gate):

- `postgres-cutover-evidence.json` from `.github/workflows/postgres-cutover-check.yml`
- Salesforce sync ops signal:
  - latest successful run from `.github/workflows/salesforce-sync-scheduler.yml`
  - latest sync run status from `/contracts/api/integrations/salesforce/sync-runs/`
  - latest webhook delivery diagnostics from `/contracts/api/integrations/webhooks/deliveries/`

## Execution

1. Trigger **Release Candidate Evidence** workflow with the release candidate tag.
2. Trigger **Postgres Cutover Check** workflow and verify artifact generation.
3. Verify Salesforce scheduler workflow success and attach run URL.
4. Verify all artifacts and attach links in PR/release notes.
5. Ensure `pr-release-evidence`, `quality-and-tenancy`, `security-scans`, and `verify-ui` checks are green before merge.
